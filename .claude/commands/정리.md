이번 대화에서 진행한 작업 내용을 `update.md`에 업데이트하고 Discord #업데이트 채널에도 알려줘.

## 순서

### 1. update.md 업데이트

- 오늘 날짜(`## YYYY-MM-DD`) 섹션을 확인
  - **섹션이 없으면**: 맨 위에 새로 추가
  - **섹션이 있으면**: 해당 섹션 아래에 내용 추가 (기존 내용 유지)
- 작업 내용을 카테고리별로 정리 (변경사항, 신규 기능, 버그 수정, 멤버 변경 등)
- 기술적인 세부사항도 포함 (어떤 파일이 왜 변경됐는지)
- 작성 후 `git add update.md && git commit && git push`까지 완료

### 2. Discord 메시지 캐시 확인

`.discord_message_cache.json` 파일에서 오늘 날짜의 message_id를 확인한다.

```bash
cat .discord_message_cache.json 2>/dev/null || echo "{}"
```

- **오늘 날짜 키가 있으면** → 기존 메시지 수정 (PATCH)
- **없으면** → 새 메시지 전송 (POST) → message_id 저장

### 3. Discord 메시지 내용 작성

update.md의 **오늘 날짜 섹션 전체**를 기반으로 메시지 작성.
(당일 2회 이상 정리 시 누적된 내용 전체가 반영되어야 함)

**메시지 형식**:
```
📦 **웹 업데이트 이력** · YYYY-MM-DD

**🆕 신규 기능**
• 항목

**🐛 버그 수정**
• 항목

**⚙️ 변경사항**
• 항목
```

- 해당 날짜에 작업한 내용만 포함 (카테고리가 없으면 생략)
- 각 항목은 간결하게 1줄로
- 기술 용어보다 사용자 관점으로 작성

### 4-A. 새 메시지 전송 (오늘 처음인 경우)

```bash
source .env && RESPONSE=$(curl -s -X POST "${DISCORD_WEBHOOK_UPDATE}?wait=true" \
  -H "Content-Type: application/json" \
  -d "{\"content\": \"<메시지 내용>\"}") && echo "$RESPONSE"
```

- `?wait=true` 필수 — 응답에서 message_id를 받기 위해
- 응답 JSON에서 `id` 필드 추출 → `.discord_message_cache.json` 업데이트

```bash
# message_id 저장 (오늘 날짜 키로)
node -e "
  const fs = require('fs');
  const cache = JSON.parse(fs.readFileSync('.discord_message_cache.json', 'utf8').trim() || '{}');
  cache['YYYY-MM-DD'] = '<message_id>';
  fs.writeFileSync('.discord_message_cache.json', JSON.stringify(cache, null, 2));
"
```

### 4-B. 기존 메시지 수정 (오늘 2회 이상인 경우)

```bash
source .env && curl -s -X PATCH "${DISCORD_WEBHOOK_UPDATE}/messages/<message_id>" \
  -H "Content-Type: application/json" \
  -d "{\"content\": \"<메시지 내용>\"}"
```

### 5. 결과 확인

- 전송/수정 성공 여부 확인
- `.discord_message_cache.json`이 올바르게 저장됐는지 확인

## 주의사항

- content 값은 JSON string escape 처리 필수 (큰따옴표 → `\"`, 줄바꿈 → `\n`)
- `.discord_message_cache.json`은 gitignore에 등록되어 있어야 함 — 없으면 `.gitignore`에 추가
- 캐시 파일이 없거나 오늘 날짜 키가 없으면 반드시 POST로 신규 전송
- Discord 메시지 최대 길이 2000자 초과 시 핵심 항목만 요약
