이번 대화에서 진행한 작업 내용을 `update.md`에 업데이트하고 Discord에도 알려줘.

## 순서

### 1. update.md 업데이트
- 오늘 날짜(`## YYYY-MM-DD`) 섹션을 맨 위에 추가 (이미 있으면 해당 섹션에 추가)
- 작업 내용을 카테고리별로 정리 (변경사항, 신규 기능, 버그 수정, 멤버 변경 등)
- 기술적인 세부사항도 포함 (어떤 파일이 왜 변경됐는지)
- 기존 내용은 건드리지 말 것
- 작성 후 git add update.md && git commit && git push까지 완료

### 2. Discord Webhook으로 업데이트 이력 전송
update.md에 작성한 내용을 바탕으로 Discord 메시지를 작성하고 전송해.

**Webhook URL**: .env 파일의 `DISCORD_WEBHOOK_UPDATE` 값 사용

**메시지 형식** (Discord 마크다운):
```
📦 **웹 업데이트 이력** · YYYY-MM-DD

**🆕 신규 기능**
• 항목

**🐛 버그 수정**
• 항목

**⚙️ 변경사항**
• 항목
```

- 해당 날짜에 작업한 내용만 포함 (카테고리가 없으면 생략)
- 각 항목은 간결하게 1줄로
- 기술 용어보다 사용자 관점으로 작성 (예: "Edge Function 배포" → "인증 시스템 업데이트")

**전송 명령어**:
```bash
source .env && curl -s -X POST "$DISCORD_WEBHOOK_UPDATE" \
  -H "Content-Type: application/json" \
  -d "{\"content\": \"<메시지 내용>\"}"
```

content 값은 JSON string escape 처리 필수 (큰따옴표 → \", 줄바꿈 → \n)
