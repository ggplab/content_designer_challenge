## 리팩토링

- **Edge Function 레이어드 구조 전환** — `discord-verify`와 `weekly-summary` 모두 controller / orchestrator / services 3계층으로 분리
  - `discord-verify`: `index.ts`(서명 검증·라우팅) → `verification.ts`(오케스트레이터) → `services/` (summarizer, sheets, discord, modal)
  - `weekly-summary`: `index.ts`(라우팅) → `summary.ts`(오케스트레이터) → `services/` (discord, gemini, sheets, url)
- **`_shared/auth.ts` 도메인별 분리** — 기존 단일 파일을 `supabase.ts` / `cors.ts` / `session.ts` / `crypto.ts` 4개로 분리, 각 Edge Function import 경로 업데이트
- **공통 유틸 `_shared/`로 통합** — `week.ts`(주차 계산), `platform.ts`(플랫폼 분류·메달), `google-auth.ts`(SA 인증), `sheets.ts`(sheetsGet/sheetsAppend), `url.ts`(shortenUrl) 추가
  - `weekly-summary/services/url.ts` 제거 → `_shared/url.ts` re-export로 통일
  - Google Sheets HTTP 요청 로직을 `_shared/sheets.ts`로 추출, discord-verify·weekly-summary 양쪽에서 공유
- **`discord-verify/service.ts` 중간 파일 제거** — controller/service 분리 1차 후 도메인별 services 2차 분리로 최종 구조 확정

## 신규 기능

- **Discord 인증 메시지 URL 단축** — is.gd API를 통해 Discord follow-up 메시지의 긴 URL을 축약 표시 (`_shared/url.ts`)

- **OG 태그 멀티라인 content 파싱 실패 수정** — 줄바꿈 허용 후 첫 줄만 추출하도록 수정 (`services/summarizer.ts`)
